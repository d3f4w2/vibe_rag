# V1 项目面试问答（病例检索 RAG）

## 1. 项目全局

### Q1：这个项目是做什么的？核心价值是什么？
A：这是一个面向宫颈阴道镜历史病例的 RAG 系统。输入病例文本后，返回 Top-K 相似病例，并输出 `HSIL | LSIL | Uncertain` 参考倾向和理由。核心价值是“可追溯辅助复核”，不替代临床最终诊断。

### Q2：为什么叫 RAG？Retrieval 和 Generation 分别是什么？
A：Retrieval 是文本向量检索（Chroma Top-K 召回相似病例）；Generation 在 v1 主要体现在基于召回证据生成解释字段（`reason`/`disclaimer`），并预留后续增强生成能力的接口。

### Q3：为什么 v1 只做文本检索，不做图像向量检索？
A：这是范围控制。文本链路更稳定、可解释、可测试；图像检索复杂度和成本更高。v1 把图像作为证据引用，不作为主召回通道，先保证主链路可靠。

### Q4：项目成功标准是什么？
A：对应 5 条验收标准：
1. 合法输入可返回 Top-K。
2. 结果可追溯到源病例。
3. 输出倾向 + 理由。
4. 稳定处理 `0x7F` 文本脏数据。
5. 证据不足必须返回 `Uncertain`。

## 2. 架构与模块

### Q5：模块边界怎么设计？
A：分为 5 层：
1. `ingestion`：扫描、清洗、时间解析、元数据。
2. `retrieval`：文档构建、向量存储、检索编排。
3. `reasoning`：倾向判定。
4. `infra`：API 访问与错误映射。
5. `cli`：参数校验、流程编排、结构化输出。
这样每层可单测、可替换、可定位故障。

### Q6：为什么把 HTTP 调用放到 `src/infra/api_client.py`？
A：避免业务层直接耦合 HTTP 细节，并统一超时/鉴权/限流/响应错误映射。这样更容易维护和替换 provider。

### Q7：为什么拆 embedding provider 和 generation provider？
A：实际场景常常是不同厂商和 endpoint。拆分后配置更清晰、容错更强，避免单配置耦合故障。

## 3. 检索与判定

### Q8：检索链路关键步骤是什么？
A：`stain_text/report_text -> 构建检索文档 -> embedding -> Chroma query Top-K -> similar_cases`。返回中包含相似度与来源元数据，保证可追溯。

### Q9：为什么有 `Uncertain`，不直接二分类？
A：医学辅助系统中“拒绝不确定”比“错误自信”更安全。证据不足时输出 `Uncertain` 可以降低误导风险。

### Q10：`Uncertain` 规则是什么？
A：
1. `top1_similarity < 0.60` 且 `majority_ratio < 0.60` 时返回 `Uncertain`。
2. Top-K 标签并列第一（tie）时返回 `Uncertain`。
3. 否则按多数标签输出 `HSIL` 或 `LSIL`。

### Q11：为什么阈值是 0.60？
A：v1 先固定阈值保证行为可预测、可测试、可复盘。它是工程安全阈，不是临床金标准，后续可基于验证集调优。

## 4. 数据质量与鲁棒性

### Q12：处理了哪些脏数据问题？
A：重点两类：
1. UTF-8 解码失败（尤其 `--stain-file`）。
2. 文本含 `0x7F` 控制字符。
策略是显式报错或清洗，不静默吞错，保证可定位。

### Q13：PDF 抽取不稳定怎么办？
A：v1 将 PDF 作为可选信息，主检索依赖 `stain_text` 可独立运行。PDF 失败不阻断主链路。

### Q14：API 超时、401、429、5xx 怎么处理？
A：统一映射为类型化异常：
`ApiTimeoutError`、`ApiAuthError`、`ApiRateLimitError`、`ApiResponseError`。CLI 再将其输出为稳定运行时错误。

## 5. 接口与工程化

### Q15：CLI 契约设计重点是什么？
A：
1. 输入校验明确（`--stain-text`/`--stain-file` 至少一个，`top-k > 0`）。
2. 输出固定 JSON 字段（`similar_cases`、`tendency`、`reason`、`disclaimer`、`meta`）。
3. 退出码稳定：成功 `0`，参数错误 `2`，运行时错误 `1`。

### Q16：怎么保证改动不破坏行为？
A：采用 TDD（先失败测试再实现），并维护验收映射（ACC-01~ACC-05）。既覆盖主链路也覆盖失败路径。

### Q17：项目里做对了哪些工程决策？
A：
1. 明确非目标，控制 v1 范围。
2. 把可追溯和不确定性处理作为一等能力。
3. 及早固定 CLI 契约、错误码与测试映射，保证可维护性。

### Q18：如果做 v2，优先做什么？
A：
1. 建立离线评估集并调优阈值。
2. 评估混合召回（向量 + 关键词）。
3. 在不破坏现有契约前提下逐步引入图像检索能力。
