# Progress Tracker（V2）

## 1. 状态与填写规则
1. 状态枚举固定为：`todo` / `ing` / `blocked` / `done` / `待提交`。
2. 未收到人类命令 `start next step` 前，不得进入下一 step。
3. `阻塞` 默认 `✗`，仅出现 blocker 时置为 `✓`，并在备注写明解除条件。
4. `已提交` 默认 `✗`，仅在人类确认完成 commit 后改为 `✓`。
5. `提交时间` 只记录 commit 时间：
   - `已提交=✓`：填写 commit 时间（ISO 或 `YYYY-MM-DD HH:mm`）。
   - `已提交=✗`：统一填写 `-`。

## 2. 当前执行步选择规则
1. 优先选择 `ing` 状态作为当前 step。
2. 若无 `ing`，优先处理 `待提交` 状态步骤（先收口再推进）。
3. 若无 `ing` 且无 `待提交`，选择 `todo` 中 `step_order` 最小的步骤。
4. 本轮文档修订属于 STEP-11 交付物优化，不视为进入 STEP-12 开发实现。
