# PRD: 宫颈阴道镜病例检索 RAG（V1 基础版）

## 1. 项目目的
基于历史病例数据构建一个可追溯的 RAG 系统。  
系统用于“找相似病例并给出分级参考倾向”，帮助人工复核，不替代临床最终诊断。

## 2. 数据结构与样本定义
当前数据按目录组织：

```text
data/
├─ HSIL/
│  └─ <case_id>/
│     ├─ 1.jpg ... 5.jpg
│     ├─ <case_id>_检查报告_<YYYYMMDDHHMMSS>.pdf
│     └─ 着色情况.txt
└─ LSIL/
   └─ <case_id>/
      ├─ 1.jpg ... 5.jpg
      ├─ <case_id>_检查报告_<YYYYMMDDHHMMSS>.pdf
      └─ 着色情况.txt
```

规则定义：
1. `label` 来自病例目录的父目录，取值 `HSIL | LSIL`。
2. 每例固定包含 `5 JPG + 1 PDF + 1 TXT`。
3. `着色情况.txt` 为 UTF-8，且可能含 `0x7F` 控制字符，需要清洗。
4. 报告时间从 PDF 文件名中的 `YYYYMMDDHHMMSS` 解析。

## 3. V1 要解决的问题
1. 输入一个病例，返回 Top-K 相似历史病例。
2. 给出可追溯证据（命中片段、来源病例、时间、路径）。
3. 输出 `HSIL | LSIL | Uncertain` 的参考倾向和简短理由。

## 4. V1 不做什么
1. 不输出临床最终诊断结论。
2. 不把图像作为主检索通道（V1 仅作证据引用）。
3. 不做自动重训练或在线学习。

## 5. 输入与输出
输入（最小集合）：
1. `case_id`（可选）
2. `stain_text`（必需，来自 `着色情况.txt`）
3. `report_text`（可选，来自 PDF 抽取）
4. `image_paths`（5 张，仅用于证据定位）
5. `top_k`（默认 5）

输出（最小集合）：
1. `similar_cases[]`：病例 ID、标签、相似度、证据片段
2. `tendency`：`HSIL | LSIL | Uncertain`
3. `reason`：基于检索证据的解释
4. `disclaimer`：仅辅助参考，不替代临床诊断

## 6. 关键流程
1. 扫描目录并构建病例清单。
2. 清洗文本（含去除 `0x7F`）。
3. 基于文本建立检索索引并执行 Top-K 检索（文本 + 可抽取 PDF 文本）。
4. 汇总检索证据，给出参考倾向与解释。
5. 输出可追溯结果，支持人工回看原始病例。

## 7. 验收标准（V1）
1. 任一合法病例输入都能返回 Top-K 相似病例。
2. 返回结果包含证据片段，并能定位到来源病例。
3. 输出包含 `HSIL | LSIL | Uncertain` 参考倾向与理由。
4. 文本清洗能稳定处理 `0x7F` 控制字符。
5. 证据不足时必须返回 `Uncertain`，不强行给结论。

## 8. 风险与后续迭代方向
1. PDF 文本抽取质量可能不稳定。  
   V1 默认允许仅使用 `着色情况.txt` 完成检索。
2. 样本类别分布可能不均衡。  
   输出中保留不确定性表达，避免过度结论。
3. 后续迭代可评估图像检索能力，但不纳入本版范围。
